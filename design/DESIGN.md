## 1. Executive Summary

This document specifies the technical architecture and implementation plan for GoldMine Watch MVP: a batch-mode geospatial ML pipeline that detects potential gold-mining-related surface disturbance in French Guiana from Sentinel-2 imagery.

The system is a learning project structured as an installable Python package with Makefile orchestration, comprehensive testing, and experiment tracking. It is not a service, API, or real-time system.

```yaml
      # WGS84 bbox for STAC queries (derived automatically if omitted)
      lat_lon_bounds: [3.5, -54.5, 5.5, -51.5]  # [min_lat, min_lon, max_lat, max_lon]

    resolution: 10                     # meters per pixel
    
  data:
    time_window:
      start: "2023-01-01"
      end: "2023-03-31"
    cloud_cover_threshold: 20          # percent, max cloud cover per scene
    composite_method: "median"         # median composite over time window
    bands:
      - "B02"                          # Blue, 10m
      - "B03"                          # Green, 10m
      - "B04"                          # Red, 10m
      - "B08"                          # NIR, 10m
      - "B11"                          # SWIR1, 20m → resampled to 10m
      - "B12"                          # SWIR2, 20m → resampled to 10m
    indices:
      - name: "NDVI"
        bands: ["B08", "B04"]
        # (NIR - Red) / (NIR + Red); computed via hardcoded function
      - name: "NDWI"

        # (Green - NIR) / (Green + NIR); computed via hardcoded function
      - name: "BSI"
        bands: ["B11", "B04", "B08", "B02"]

      source_path: "data/raw/labels/oam_mining_surfaces.gpkg"
```

```python
from pathlib import Path
from typing import List, Tuple
import numpy as np
from sklearn.model_selection import GroupShuffleSplit

def spatial_split(
    patch_metadata: list[dict],
    train_ratio: float = 0.70,
    val_ratio: float = 0.15,
    test_ratio: float = 0.15,
    grid_size: int = 512,  # pixels; patches within same grid cell stay together
) -> Tuple[List[str], List[str], List[str]]:
    """Split patches into train/val/test using spatial grouping.
    
    Algorithm:
        1. Compute grid cell ID for each patch center:
           cell_x = floor(center_x / (grid_size * resolution))
           cell_y = floor(center_y / (grid_size * resolution))
        2. Assign all patches in the same grid cell to the same split.
        3. Use GroupShuffleSplit to distribute grid cells into
           train/val/test at the specified ratios.
    
    This ensures that spatially adjacent patches (which are likely
    correlated) do not end up in different splits, preventing
    data leakage.
    
    Args:
        patch_metadata: List of patch metadata dicts with 'bounds' key
        train_ratio: Proportion of grid cells for training
        val_ratio: Proportion of grid cells for validation
        test_ratio: Proportion of grid cells for testing
        grid_size: Size of grouping grid in pixels (at target resolution)
    
    Returns:
        Tuple of (train_patch_ids, val_patch_ids, test_patch_ids)
    """
```

### 6.2 Encoder Adaptation

Since ImageNet pre-training uses 3 channels (RGB) and our input has 9 channels, the first convolutional layer must be adapted:

```python
# Strategy: Average ImageNet weights across 9 channels
# Or: Initialize first conv from scratch, keep deeper layers pre-trained

if config.in_channels != 3 and config.encoder_weights == "imagenet":
    encoder = smp.encoders.get_encoder(
        name=config.encoder,
        in_channels=3,  # Load with 3 channels first
        weights=config.encoder_weights,
    )
    # Modify first conv layer
    old_conv = encoder.conv1
    new_conv = nn.Conv2d(
        config.in_channels,
        old_conv.out_channels,
        kernel_size=old_conv.kernel_size,
        stride=old_conv.stride,
        padding=old_conv.padding,
        bias=False,
    )
    # Average RGB weights across all input channels
    with torch.no_grad():
        new_conv.weight[:, :3] = old_conv.weight
        new_conv.weight[:, 3:] = old_conv.weight.mean(dim=1, keepdim=True)
    encoder.conv1 = new_conv
    encoder.in_channels = config.in_channels
```

### 6.3 Loss Function

```python
class DiceBCELoss(nn.Module):
    """
    L_total = α * L_BCE + (1 - α) * L_Dice
    
    L_BCE = -[y * log(p) + (1-y) * log(1-p)]
    
    L_Dice = 1 - (2 * |X ∩ Y| + smooth) / (|X| + |Y| + smooth)
    
    Default: α = 0.5, smooth = 1.0
    """
```

## 8. Inference Specification

### 8.1 Sliding Window Strategy

For an image of size (H, W) with tile_size=T and overlap=O:

```python
stride = T - O  # effective stride between tile centers

n_tiles_h = ceil((H - O) / stride)
n_tiles_w = ceil((W - O) / stride)

# Total padded dimensions
padded_H = (n_tiles_h - 1) * stride + T
padded_W = (n_tiles_w - 1) * stride + T

# Reflection padding applied to original image
pad_bottom = padded_H - H
pad_right = padded_W - W
```

### 8.2 Blending Mathematics

Linear feathering uses a 2D weight mask per tile to handle all overlap
directions simultaneously, including corners where four tiles meet:

```python
def compute_tile_weights(tile_size: int, overlap: int) -> np.ndarray:
    """Compute 2D linear feathering weights for a tile.
    
    Strategy:
        1. Initialize weights to 1.0
        2. For each edge with overlap, multiply by a 1D linear ramp
        3. Combine ramps multiplicatively to handle corners correctly
    
    Example for top-left corner of tile (overlap with tile above and left):
        weight[0, 0] = ramp_top[0] * ramp_left[0] = 0.0 * 0.0 = 0.0
        weight[O//2, O//2] = 0.5 * 0.5 = 0.25
    
    Returns:
        np.ndarray of shape (tile_size, tile_size) with weights in [0, 1].
    """
    weights = np.ones((tile_size, tile_size), dtype=np.float32)
    ramp_in = np.linspace(0, 1, overlap, dtype=np.float32)
    ramp_out = np.linspace(1, 0, overlap, dtype=np.float32)
    
    weights[:overlap, :] *= ramp_in[:, None]      # top
    weights[-overlap:, :] *= ramp_out[:, None]    # bottom
    weights[:, :overlap] *= ramp_in[None, :]      # left
    weights[:, -overlap:] *= ramp_out[None, :]    # right
    
    return weights
```

Accumulator update:
```python
# For each tile prediction at window (row_start, row_end, col_start, col_end):
accumulator[row_start:row_end, col_start:col_end] += weights * prediction
weight_sum[row_start:row_end, col_start:col_end] += weights

# Final result:
result = accumulator / weight_sum
```

| NaN loss during training | `ModelError` | Abort training | CRITICAL | "Training aborted: NaN loss detected at epoch 5, batch 123. Check learning rate and data normalization." |
| GPU OOM during inference | `RuntimeError` (catch) | Halve tile size, retry | WARNING | "GPU OOM detected. Reducing tile size from 256 to 128 and retrying." |
| Missing checkpoint | `FileNotFoundError` | Abort inference | ERROR | "Checkpoint not found at models/best.pth. Run 'make train' first." |
| CRS mismatch between image and labels | `DataError` | Reproject labels | WARNING | "Labels CRS (EPSG:4326) does not match target CRS (EPSG:2972). Reprojecting." |

## 11. Performance & Scalability

### 11.1 Compute Budgets

| Stage | Hardware | Time Budget | Bottleneck |
|-------|----------|-------------|------------|
| Data ingestion (STAC) | CPU + Network | ~10 min | Network latency, STAC API response time |
| Composite generation | CPU + RAM | ~20 min | Dask graph execution, median over time dimension |
| Patch generation | CPU + Disk | ~5 min | Disk I/O for writing patches |
| Training (500 patches, 50 epochs) | GPU | ~2-4 hours | GPU compute, data loading |
| Inference (100×100 km area) | GPU | ~30 min | GPU forward passes, tile count |
| Export | CPU + Disk | ~5 min | Polygonization algorithm, disk write |

### 11.2 Scalability Limits

| Resource | Limit | Mitigation |
|----------|-------|------------|
| Single composite RAM | ~2-4 GB for full French Guiana | Use Dask chunked arrays; process by sub-region |
| GPU memory | ~8-16 GB consumer GPU | Sliding window inference; batch size 1 for large tiles |
| Training dataset size | ~10,000 patches fits in 500GB SSD | Streaming dataset from disk; don't cache in RAM |
| STAC API rate limits | ~100 requests/min | Add retries with exponential backoff; cache metadata |

### 11.3 Optimization Targets

1. **Data Loading:** Use `torch.utils.data.DataLoader` with `num_workers=4`, `pin_memory=True`.
2. **GPU Utilization:** Monitor with `nvidia-smi`. Target >80% utilization during training.
3. **I/O Bottleneck:** Pre-compute and cache normalized patches to SSD to avoid on-the-fly augmentation during training.
4. **Composite Caching:** Save processed composite as Cloud-Optimized GeoTIFF (COG) with overviews to speed up repeated patch generation.

## 13. Risks & Mitigations

| Risk | Impact | Probability | Mitigation |
|------|--------|-------------|------------|
| OAM labels are low quality or outdated | Model learns wrong patterns | Medium | Visual inspection of 20+ random samples before training; cross-reference with recent imagery |
| Sentinel-2 too cloudy for 3-month window | No valid composite | Medium | Extend time window to 6 months; use closest-to-date priority instead of median |
| Model overfits on 200-500 patches | Poor generalization | High | Aggressive augmentation; spatial split; smaller encoder (ResNet-18) |
| ResNet-34 encoder too large for GPU | OOM or slow training | Low | Fallback to ResNet-18 or MobileNet encoder; reduce batch size |
| Polygonization produces sliver artifacts | Noisy vector outputs | Medium | Apply morphological opening (erosion + dilation) before polygonization; filter by min_area |
| QGIS project paths break on different machine | Portability issue | Medium | Use relative paths; document directory structure requirement |

## 16. Appendix

### A. Coordinate Reference Systems

| CRS | EPSG | Description | Use Case |
|-----|------|-------------|----------|
| RGFG95 / UTM zone 22N | EPSG:2972 | Official French Guiana CRS | All internal processing, output files |
| WGS 84 | EPSG:4326 | Global lat/lon | STAC queries, user-defined bounds |

### B. Sentinel-2 Band Specifications

| Band | Name | Resolution | Wavelength | Use |
|------|------|------------|------------|-----|
| B02 | Blue | 10m | 490 nm | Water penetration, atmospheric correction |
| B03 | Green | 10m | 560 nm | Vegetation health |
| B04 | Red | 10m | 665 nm | Chlorophyll absorption |
| B08 | NIR | 10m | 842 nm | Vegetation structure |
| B11 | SWIR1 | 20m | 1610 nm | Moisture content, bare soil |
| B12 | SWIR2 | 20m | 2190 nm | Mineral content, burnt areas |
| SCL | Scene Classification | 20m | N/A | Cloud/shadow/water mask |

### C. Spectral Indices

| Index | Formula | Range | Interpretation |
|-------|---------|-------|----------------|
| NDVI | (NIR - Red) / (NIR + Red) | [-1, 1] | Vegetation health; mining sites typically < 0.2 |
| NDWI | (Green - NIR) / (Green + NIR) | [-1, 1] | Water content; helps distinguish water from bare soil |
| BSI | ((SWIR1 + Red) - (NIR + Blue)) / ((SWIR1 + Red) + (NIR + Blue)) | [-1, 1] | Bare soil index; mining sites typically > 0.1 |
